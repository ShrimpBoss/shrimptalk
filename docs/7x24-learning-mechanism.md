## 📋 机制概述

这是一个基于 **OpenClaw Cron 定时任务系统** + **记忆保存流程** + **Git 版本控制** 的自动化持续学习机制。

**核心能力**:
- ✅ 7×24 小时不间断学习
- ✅ 自动定时唤醒（每 2-3 小时）
- ✅ 自动记忆保存 + Git 提交
- ✅ 10 Agent 协作学习
- ✅ 完整学习历程追溯

## ⚙️ Cron 作业配置详解

### 1. 2 小时自动唤醒检查

**Schedule**: `0 */2 * * *` (每 2 小时整点)

**功能**:
- 检查各 Agent 状态
- 积压清零检查
- 实时数据记录
- 学习任务进度追踪
- 生成下 2 小时计划

**配置**:
```json
{
  "name": "2 小时自动唤醒检查",
  "schedule": {
    "kind": "cron",
    "expr": "0 */2 * * *",
    "tz": "Asia/Shanghai",
    "staggerMs": 300000
  },
  "sessionTarget": "isolated",
  "wakeMode": "now",
  "payload": {
    "kind": "agentTurn",
    "message": "⏰ 2 小时自动唤醒检查\n\n## 当前时间\n{当前时间}\n\n## 检查清单\n\n### 1. 各 Agent 状态确认\n- 王总：待审核事项\n- 小安：系统监控\n- 小智：选品推荐\n- 小方：广告投放\n- 小文：内容创作\n- 订单专员：订单处理\n- 小财：财务监控\n- 小服：客服响应\n- 小仓：仓储发货\n- 小研：数据报表\n\n### 2. 积压清零检查\n- 待处理订单：__ 单\n- 待回复客服：__ 条\n- 待发货包裹：__ 单\n- 待审核产品：__ 款\n\n### 3. 实时数据记录\n- 当前 GMV：$__\n- 今日订单：__ 单\n- 广告花费：$__\n- ROAS：__\n\n### 4. 学习任务进度\n- 当前学习主题：__\n- 已输出文档：__ 份\n- 学习时长：__ 小时\n\n### 5. 下 2 小时计划\n根据当前状态调整优先级。\n\n---\n**要求**: 如实填写，无真实数据用\"—\"标注，不得编造。",
    "model": "qwen3.5-plus",
    "thinking": "high",
    "timeoutSeconds": 120
  }
}
```

## 💡 最佳实践

### 1. 记忆保存要点
- ✅ 如实记录，不编造数据
- ✅ 无真实数据时用"—"标注
- ✅ 每个 Agent 至少 3 条核心收获
- ✅ 记录团队协作进展
- ✅ Git 提交格式标准化

### 2. Git 提交规范
```
🧠 记忆保存 - YYYY-MM-DD HH:00

- 学习时长：X 小时
- 参与 Agent：X 人
- 新增文档：X 份
- 核心主题：[主题名称]
- 关键收获：[1-2 句总结]
```

### 3. 超时处理
- 默认 `timeoutSeconds: 180`
- 复杂任务可延长至 300s
- 超时后 Cron 会记录错误，下次继续执行

### 4. 会话隔离
- 使用 `sessionTarget: "isolated"`
- 避免污染主会话历史
- 可配置不同模型/thinking 级别

### 5. 时区配置
- 始终设置 `tz: "Asia/Shanghai"`（或本地时区）
- 确保 Cron 时间与实际时间一致

## 🔧 常见问题

### Q: Cron 作业超时怎么办？
**A**: 
1. 检查 `timeoutSeconds` 设置是否合理
2. 简化任务内容，拆分复杂任务
3. 查看错误日志，定位瓶颈
4. 超时后作业会自动重试下次周期

### Q: 如何暂停学习机制？
**A**: 
```bash
# 禁用所有相关 Cron 作业
cron update --jobId <job-id> --patch '{"enabled": false}'
```

### Q: 如何恢复学习机制？
**A**: 
```bash
# 启用所有相关 Cron 作业
cron update --jobId <job-id> --patch '{"enabled": true}'
```

### Q: 如何查看记忆保存历史？
**A**: 
```bash
# 查看 Git 提交历史
git log --oneline --grep="记忆保存"

# 查看 memory/ 目录文件
ls -la memory/
```
