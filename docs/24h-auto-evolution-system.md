## 📊 OA 指标体系

### 核心指标 (Built-in)

#### 1. Cron 可靠性
- **指标**: 成功率、准时率、连续失败次数
- **来源**: `~/.openclaw/cron/runs/*.jsonl`
- **健康线**: >95%
- **告警线**: <80%
- **自动修复**: 失败 3 次→自动调整 timeout→记录修复

#### 2. Agent 健康度
- **指标**: 活跃 Agent 数、Memory 记录率、平均响应时间
- **来源**: `~/.openclaw/sessions/` + Memory 文件
- **健康线**: 10/10 在线，>90% Memory 记录
- **告警线**: <8/10 在线
- **自动修复**: 离线 Agent→自动重启→记录原因

#### 3. Memory 纪律
- **指标**: Memory 文件大小、更新频率、冗余度
- **来源**: `MEMORY.md` + `memory/*.md`
- **健康线**: <50KB，每日更新
- **告警线**: >100KB，3 天未更新
- **自动修复**: 自动归档旧内容→提炼核心→压缩

### 自定义指标 (Custom)

#### 4. 知识飞轮
- **指标**: 新增文档数、核心收获数、复用次数
- **来源**: `learning/` + `memory/` 目录
- **目标**: 每日 3+ 文档，每周 1 次回顾

#### 5. 变现进展
- **指标**: 技能包数量、访问量、转化率
- **来源**: GitHub Pages 访问统计 (待接入)
- **目标**: 每周 1 个新技能包

#### 6. 系统进化
- **指标**: 自动修复次数、Prompt 优化次数、效率提升
- **来源**: `docs/` 修复记录 + Git 提交历史
- **目标**: 每周 2+ 自动修复，每月 1 次大优化

## 🛠️ 自动修复机制

### Cron 作业失败自动修复

```python
# 伪代码逻辑
if cron_job.consecutive_errors >= 3:
    # 分析失败原因
    if error_type == "timeout":
        # 自动延长 timeout
        new_timeout = current_timeout * 1.5
        update_cron_job(timeout=new_timeout)
        log_fix("自动延长 timeout", old=current, new=new_timeout)
    
    elif error_type == "channel_required":
        # 自动改为 mode:none
        update_cron_job(delivery_mode="none")
        log_fix("自动关闭 delivery", job=job_id)
    
    elif error_type == "memory_full":
        # 自动归档旧 Memory
        archive_old_memory()
        log_fix("自动归档 Memory")
    
    # 重置错误计数
    reset_error_count()
    
    # 记录修复历史
    save_to_evolution_log()
```

### Agent 离线自动重启

# 伪代码逻辑
