## 🎨 3D 形象设计

### 虾老板角色设计

**基本特征**:
```
🦐 虾老板 (Shrimp Boss)
├── 身体：粉红色半透明虾身 (高 1.2m)
├── 眼睛：大眼睛 + 圆框眼镜 (显示表情)
├── 配件：红色小领结
├── 触须：2 根灵活触须（用来打字/翻书）
└── 尾巴：扇形尾巴（表达情绪时摆动）
```

**表情系统**:
| 心情 | 眼睛 | 嘴巴 | 触须 | 尾巴 |
|------|------|------|------|------|
| 😊 开心 | 弯弯的 | 微笑 | 放松摆动 | 轻快摇摆 |
| 🤔 专注 | 认真 | 抿嘴 | 静止 | 微微摆动 |
| 😫 累了 | 半闭 | 下垂 | 无力下垂 | 缓慢摆动 |
| 😴 困了 | 闭着 | 微张 | 静止 | 静止 |
| 😵 阻塞 | 转圈 | 波浪线 | 乱挥 | 快速摆动 |
| 🤓 学习 | 闪光 | 微笑 | 拿笔/书 | 兴奋摇摆 |

## 🎬 动画设计

### 虾老板动作库

| 动作 | 触发条件 | 动画描述 |
|------|---------|---------|
| **💻 打字** | 内容创作/编程时 | 触须在键盘上快速移动 |
| **📚 看书** | 学习时 | 触须翻书，眼睛跟随文字 |
| **🤔 思考** | 深度思考时 | 触须摸下巴，眼睛向上看 |
| **🧘 冥想** | 休息时 | 盘腿坐，触须放膝盖，闭眼 |
| **😴 睡觉** | 凌晨 0-6 点 | 躺在床上，有呼吸起伏 |
| **🎮 玩耍** | 空闲时 | 玩小游戏/看电视 |
| **📊 分析** | 数据复盘时 | 看图表，触须点屏幕 |
| **📧 回复** | 用户互动时 | 打字 + 表情变化 |
| **🎨 创作** | 创意时间 | 画画/写剧本动作 |
| **😵 阻塞** | 任务卡住时 | 挠头 + 转圈眼睛 |

### 场景切换动画

```
办公室 → 卧室：虾老板走向门，淡出，卧室淡入
卧室 → 书房：虾老板走向书架，场景旋转切换
书房 → 客厅：虾老板放下书，走向沙发
```

## 🛠️ 技术实现方案

### 方案 A: Three.js + Blender（推荐）

**工作流**:
```
Blender 建模 → 导出 GLTF → Three.js 加载 → 动画控制
```

**优点**:
- 完整 3D 能力
- 支持骨骼动画
- 成熟生态
- 性能好

**开发时间**: 5-7 天

### 方案 B: Spline（快速版）

**工作流**:
```
Spline 设计 → 导出代码 → 嵌入网页
```

**优点**:
- 设计友好
- 快速出效果
- 无需建模基础

**开发时间**: 2-3 天

### 方案 C: Ready Player Me + Three.js（混合）

**工作流**:
```
RPM 创建角色 → Three.js 场景 → 自定义动画
```

**优点**:
- 角色质量高
- 快速创建
- 可定制

**开发时间**: 3-4 天
