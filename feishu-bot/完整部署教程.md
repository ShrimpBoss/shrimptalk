## 5. 配置安全设置

**强烈建议配置安全设置！** 防止 Webhook 泄露后被恶意调用。

### 5.1 方式一：签名校验（最安全）⭐ 推荐

#### 步骤 1：启用签名校验

1. **在机器人配置页面**
   ```
   找到 「安全设置」区域
   ```

2. **选择「签名校验」**
   ```
   点击 「签名校验」选项
   或勾选复选框
   ```

3. **查看 Secret**
   ```
   系统会生成一个 Secret：
   SECxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   
   格式：SEC + 32 位随机字符
   ```

#### 步骤 2：复制 Secret

1. **点击「复制」按钮**
   ```
   旁边有 「复制」或 📋 图标
   点击它
   ```

2. **验证已复制**
   ```
   粘贴到文本编辑器
   确认完整复制（包含 SEC 前缀）
   ```

3. **⚠️ 重要提醒**
   ```
   - Secret 和 Webhook 一样重要
   - 两者结合才能发送消息
   - 分开保管更安全
   ```

#### 步骤 3：保存配置

1. **点击「保存」按钮**
   ```
   页面底部或右上角
   点击 「保存」或 「完成」
   ```

2. **确认保存成功**
   ```
   显示提示：「保存成功」
   ✅ 签名校验已启用
   ```

### 5.2 方式二：IP 白名单（适合固定 IP）

#### 步骤 1：获取你的服务器 IP

```bash
# 在服务器上运行
curl ifconfig.me
# 或
curl ipinfo.io/ip
```

记录返回的 IP 地址，例如：`123.45.67.89`

#### 步骤 2：配置 IP 白名单

1. **在机器人配置页面**
   ```
   找到 「安全设置」区域
   ```

2. **选择「IP 白名单」**
   ```
   点击 「IP 白名单」选项
   ```

3. **添加 IP 地址**
   ```
   输入你的服务器 IP：123.45.67.89
   
   支持格式：
   - 单个 IP：123.45.67.89
   - IP 段：123.45.67.*
   - CIDR：123.45.67.0/24
   ```

4. **最多添加 10 个 IP**
   ```
   如果有多个服务器，用回车分隔
   ```

5. **点击「保存」**
   ```
   ✅ IP 白名单已配置
   ```

### 5.3 方式三：自定义关键词（最简单）

#### 步骤 1：设置关键词

1. **在机器人配置页面**
   ```
   找到 「安全设置」区域
   ```

2. **选择「自定义关键词」**
   ```
   点击 「自定义关键词」选项
   ```

3. **添加关键词**
   ```
   输入关键词，每个一行：
   Dashboard
   报警
   通知
   系统
   ```

4. **最多 10 个关键词**
   ```
   消息内容必须包含至少一个关键词
   才能发送成功
   ```

5. **点击「保存」**
   ```
   ✅ 自定义关键词已配置
   ```

### 5.4 安全设置对比

| 方式 | 安全性 | 难度 | 适用场景 |
|------|--------|------|----------|
| **签名校验** | ⭐⭐⭐⭐⭐ | 中 | 生产环境（推荐） |
| **IP 白名单** | ⭐⭐⭐⭐ | 低 | 固定 IP 服务器 |
| **关键词** | ⭐⭐⭐ | 低 | 简单测试 |

**最佳实践**：签名校验 + IP 白名单（双重保护）

## 9. 配置定时通知

### 9.1 创建定时任务脚本

创建文件：`~/.openclaw/workspace/feishu-bot/daily-report.js`

```javascript
#!/usr/bin/env node
/**
 * 每日经营日报 - 定时发送
 * 每天上午 9:00 自动发送
 */

const FeishuNotifier = require('./feishu_integration');
const notifier = new FeishuNotifier();

// 模拟获取今日数据（实际应从数据库读取）
async function getTodayMetrics() {
  // TODO: 从 Dashboard 数据库获取真实数据
  return {
    gmv: 1234,
    orders: 45,
    profit: 487,
    roas: 3.2
  };
}

async function sendDailyReport() {
  console.log('📊 发送每日经营日报...');
  
  const metrics = await getTodayMetrics();
  const success = await notifier.sendDailyReport(metrics);
  
  if (success) {
    console.log('✅ 日报发送成功');
  } else {
    console.error('❌ 日报发送失败');
  }
}

// 立即执行一次（测试用）
sendDailyReport();

// 定时任务：每天 9:00 发送
// 生产环境建议使用 cron 或系统定时任务
```

## 10. 故障排查

### 问题 1：收不到消息

**检查清单**：
- [ ] Webhook 地址是否正确
- [ ] 机器人是否在群内
- [ ] 网络是否通畅
- [ ] 是否被限流（100 次/分钟）

**测试方法**：
```bash
curl -X POST https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_WEBHOOK \
  -H "Content-Type: application/json" \
  -d '{"msg_type":"text","content":{"text":"test"}}'
```

### 问题 2：签名校验失败

**错误信息**：
```json
{"code":19021,"msg":"sign match fail or timestamp is not within one hour from current time"}
```

**原因**：
1. Secret 错误
2. 时间戳超过 1 小时
3. 服务器时间不准确

**解决方法**：
```bash
# 校准服务器时间
# Linux
sudo ntpdate -s time.nist.gov

# macOS
sudo sntp -sS time.apple.com

# 验证时间
date
```

### 问题 3：IP 白名单失败

**错误信息**：
```json
{"code":19022,"msg":"Ip Not Allowed"}
```

**原因**：服务器 IP 不在白名单内

**解决方法**：
1. 获取当前 IP：`curl ifconfig.me`
2. 在飞书机器人配置中添加该 IP
3. 保存并重试

### 问题 4：关键词不匹配

**错误信息**：
```json
{"code":19024,"msg":"Key Words Not Found"}
```

**原因**：消息内容不包含任何关键词

**解决方法**：
1. 在消息中添加至少一个关键词
2. 或修改机器人配置的关键词列表

### 问题 5：频率限制

**错误信息**：
```json
{"code":9499,"msg":"Too Many Requests"}
```

**原因**：超过 100 次/分钟 或 5 次/秒

**解决方法**：
1. 降低发送频率
2. 避开整点/半点高峰时段
3. 合并多条消息为一条

| 类型 | msg_type | 说明 |
|------|----------|------|

await notifier.sendText('系统运行正常');

await notifier.sendPost('通知标题', [
  [
