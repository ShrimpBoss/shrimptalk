**库位规划原则**
- ABC 分类：A 类爆款靠近打包区，C 类滞销放远
- 同类集中：同类商品集中存放，减少走动
- 重物下层：重物放下层，轻物放上层
- 通道合理：主通道 1.5-2 米，次通道 0.8-1 米

### 2.2 库存管理模型

**ABC 分类法**
| 分类 | 标准 | 占比 | 管理策略 |
|------|------|------|----------|
| A 类 | 销量前 20% SKU | 70% 销量 | 重点管理、安全库存高、盘点频繁 |
| B 类 | 销量中间 30% SKU | 20% 销量 | 常规管理、安全库存中、盘点定期 |
| C 类 | 销量后 50% SKU | 10% 销量 | 简化管理、安全库存低、盘点减少 |

**安全库存公式**
```
安全库存 = (最大日销量 × 最大补货天数) - (平均日销量 × 平均补货天数)

示例:
- 最大日销量：100 件
- 平均日销量：60 件
- 最大补货天数：20 天（生产 15 天 + 运输 5 天）
- 平均补货天数：15 天

安全库存 = (100 × 20) - (60 × 15) = 2000 - 900 = 1100 件

补货点 = 安全库存 + (平均日销量 × 平均补货天数)
补货点 = 1100 + (60 × 15) = 1100 + 900 = 2000 件

当库存降至 2000 件时，立即下单补货
```

**库存周转率**
```
库存周转率 = 销售成本 / 平均库存
库存周转天数 = 365 / 库存周转率

健康标准:
- 优秀：<30 天
- 良好：30-45 天
- 一般：45-60 天
- 预警：>60 天
```

### 2.4 海外仓模式分析

**海外仓类型**
| 类型 | 说明 | 优点 | 缺点 | 适合 |
|------|------|------|------|------|
| 平台仓 | Amazon FBA | 流量扶持、配送快 | 费用高、库存风险 | 平台卖家 |
| 第三方仓 | 3PL 海外仓 | 灵活、多平台 | 管理复杂 | 多平台卖家 |
| 自建仓 | 自己租赁仓库 | 完全控制 | 投入大、风险高 | 大型卖家 |

**海外仓成本结构**
```
海外仓成本
├── 头程物流：¥30-50/kg（中国到海外仓）
├── 入库费：$0.1-0.3/件
├── 仓储费：$5-15/立方/月
├── 订单处理费：$2-5/单
├── 尾程配送：$5-15/单（USPS/UPS/FedEx）
└── 退货处理：$3-10/单

总成本估算：
- 小件（<500g）：$8-12/单
- 中件（500g-2kg）：$10-18/单
- 大件（>2kg）：$15-30/单
```

**海外仓决策模型**
```
适合海外仓的情况:
✓ 日销稳定在 50 单以上
✓ 产品生命周期>6 个月
✓ 毛利空间>40%
✓ 产品标准化、不易过时
✓ 平台有流量扶持（如 Amazon）

不适合海外仓的情况:
✗ 销量不稳定
✗ 产品更新快、易过时
✗ 毛利低（<30%）
✗ 体积大、仓储成本高
✗ 合规风险高
```

### 2.5 包装优化策略

**包装成本构成**
| 材料 | 成本 | 优化空间 |
|------|------|----------|
| 外箱 | ¥2-5/个 | 尺寸优化、批量采购 |
| 填充物 | ¥0.5-2/个 | 减少用量、替代材料 |
| 胶带 | ¥0.3-0.5/卷 | 自动化、减少缠绕 |
| 面单 | ¥0.2-0.5/张 | 电子面单、批量打印 |
| 人工 | ¥1-3/单 | 流程优化、自动化 |

**体积重优化**
```
体积重公式:
- 国际快递：长×宽×高 (cm) / 5000 = 体积重 (kg)
- 专线物流：长×宽×高 (cm) / 6000 = 体积重 (kg)

计费重量 = max(实际重量，体积重量)

优化案例:
原包装：30×20×15cm = 9000/6000 = 1.5kg 体积重
优化后：25×18×12cm = 5400/6000 = 0.9kg 体积重
节省：0.6kg × ¥60/kg = ¥36/单
```

## 3. 实操指南

### 3.1 仓库日常操作流程

**收货流程**
```
Step 1: 预约到货
- 供应商提前 24 小时预约
- 安排收货人员与库位

Step 2: 卸货验货
- 核对送货单与采购单
- 抽检商品质量（AQL 2.5）
- 拍照留底（外箱、商品）

Step 3: 系统入库
- 扫描商品条码
- 系统分配库位
- 打印库位标签

Step 4: 上架存放
- 按库位标签上架
- ABC 分类存放
- 更新库存系统
```

**拣货流程**
```
Step 1: 订单汇总
- ERP 系统导出待拣货订单
- 按库位排序生成拣货单
- 波次拣货（多订单合并）

Step 2: 拣货作业
- 按拣货单路径拣货
- 扫描商品条码核对
- 放入拣货篮/车

Step 3: 复核打包
- 复核商品与订单一致
- 选择合适包装
- 打包贴单

Step 4: 发货交接
- 按物流商分拣
- 扫描揽收
- 系统更新发货状态
```

**盘点流程**
```
每日动盘:
- 盘点当日有变动的 SKU
- 核对系统库存与实物

每周抽盘:
- 随机抽取 20% SKU 盘点
- A 类 SKU 每周必盘

每月全盘:
- 全部 SKU 盘点
- 差异分析与调整

每季大盘:
- 全面盘点 + 审计
- 年度库存报告
```

### 3.2 物流商谈判要点

**谈判准备**
- 收集 3-5 家物流商报价
- 统计月发货量、目的地分布
- 分析当前物流问题与痛点
- 确定谈判目标（价格、时效、服务）

**谈判要点**
```
价格谈判:
- "我们月发货量 XXX，能否给到更优惠价格？"
- "如果签约 1 年，能否享受 volume discount？"
- "竞争对手报价 XXX，能否 match？"

时效保障:
- "能否保证 X 天内妥投率>95%？"
- "延误如何赔付？"
- "旺季能否保证舱位？"

服务承诺:
- "客服响应时间？"
- "理赔流程与时效？"
- "系统对接支持？"
```

**合同关键条款**
- 价格条款（首重、续重、附加费）
- 时效条款（承诺时效、延误赔付）
- 理赔条款（破损、丢失、赔付标准）
- 结算条款（月结、账期）
- 终止条款（解约条件、通知期）

### 3.3 库存预警机制

**预警级别**
| 级别 | 条件 | 响应 |
|------|------|------|
| 绿色 | 库存>补货点 | 正常 |
| 黄色 | 安全库存<库存<补货点 | 准备补货 |
| 橙色 | 库存<安全库存 | 立即补货 |
| 红色 | 库存<7 天销量 | 紧急补货 + 限购 |

**预警流程**
```
系统自动监控:
- 每日更新库存数据
- 自动计算补货点
- 触发预警自动通知

人工响应:
- 黄色预警：小智评估，3 天内决策
- 橙色预警：立即联系供应商，1 天内决策
- 红色预警：王总审批，紧急处理

跟进闭环:
- 补货订单跟踪
- 预计到货时间更新
- 到货入库确认
```

### 3.4 海外仓运营流程

**头程发货**
```
Step 1: 补货计划
- 根据销量预测确定补货量
- 考虑海运时效（30-45 天）
- 提前 45 天启动补货

Step 2: 订舱发货
- 联系货代订舱
- 准备报关资料
- 货物入仓、装柜

Step 3: 清关派送
- 目的港清关
- 派送至海外仓
- 海外仓收货上架
```

**海外仓管理**
```
库存监控:
- 每日同步库存数据
- 预警低库存 SKU
- 呆滞库存处理

订单处理:
- 订单自动同步海外仓
- 海外仓拣货打包
- 尾程配送

退货处理:
- 退货入海外仓
- 检验可二次销售
- 上架或销毁
```

## 5. 案例分享

### 5.1 成功案例：某卖家仓库优化

**背景**：日订单 500 单，仓库效率低

**问题**：
- 拣货路径混乱，效率低
- 库存不准，经常超卖
- 打包慢，发货不及时

**优化措施**：
- 重新规划库位（ABC 分类）
- 引入 PDA 扫描，减少错误
- 优化拣货路径（S 型路线）
- 设置打包流水线

**效果**：
- 拣货效率提升 50%
- 库存准确率 99.5%
- 24 小时发货率 98%

### 5.2 成功案例：某卖家海外仓布局

**背景**：Amazon 卖家，日销 200 单

**决策**：
- 爆款入 FBA（流量扶持）
- 长尾款自发货（降低风险）
- 美国东西两岸各一仓（降低尾程）

**效果**：
- 配送时效 2-3 天
- 运费降低 30%
- 销量提升 50%

### 5.3 失败案例：某卖家库存积压

**背景**：备货 10000 件，销量不及预期

**问题**：
- 过度乐观预测
- 未设置安全库存
- 未及时促销清仓

**后果**：
- 库存积压 8000 件
- 仓储费每月$5000
- 最终清仓亏损 50 万

**教训**：
- 备货要保守，宁缺毋滥
- 建立安全库存模型
- 滞销及时清仓
