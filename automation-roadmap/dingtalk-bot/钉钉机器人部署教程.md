## 1. 钉钉机器人简介

### 什么是钉钉自定义机器人？

钉钉自定义机器人是一个可以通过 Webhook 地址向钉钉群聊发送消息的机器人。

### 支持的消息类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| text | 文本消息 | 简单通知 |
| markdown | Markdown 消息 | 格式化报告 |
| link | 链接消息 | 文章分享 |
| actionCard | 卡片消息 | 交互式通知 |
| feedCard | 动态消息 | 信息流 |

### 与飞书对比

| 特性 | 钉钉 | 飞书 |
|------|------|------|
| Webhook | ✅ | ✅ |
| 签名校验 | ✅ | ✅ |
| 关键词 | ✅ | ✅ |
| IP 白名单 | ❌ | ✅ |
| Markdown | ✅ | ✅ |
| 卡片消息 | ✅ | ✅ |

## 2. 添加机器人到群聊

### 2.1 打开钉钉

1. **下载并安装钉钉**
   - 官网：https://www.dingtalk.com
   - 或使用网页版

2. **登录账号**
   - 使用手机号或钉钉账号登录

### 2.2 创建或选择群聊

**选项 A：使用现有群**
- 工作群、项目群等
- 确保你是群主或管理员

**选项 B：创建新群**
1. 点击左上角「+」→「发起群聊」
2. 选择「普通群」
3. 添加成员（可以先只加自己）
4. 设置群名称：Dashboard 通知群

### 2.3 添加机器人

1. **进入群设置**
   ```
   在群聊界面 → 点击右上角「≡」或「...」
   ```

2. **找到「群机器人」**
   ```
   向下滑动 → 找到「群机器人」
   点击进入
   ```

3. **点击「添加机器人」**
   ```
   点击底部或右上角的「+ 添加机器人」
   ```

4. **选择「自定义」机器人**
   ```
   在机器人列表中找到「自定义」
   图标通常是 🤖
   点击「添加」
   ```

5. **配置机器人信息**
   ```
   机器人名称：Dashboard 助手
   机器人头像：可选
   ```

6. **点击「完成」**
   ```
   ✅ 机器人添加成功！
   ```

## 3. 获取 Webhook 地址

### 3.1 进入机器人配置

1. **在群机器人列表**
   ```
   找到你刚添加的「Dashboard 助手」
   点击它
   ```

2. **查看机器人详情**
   ```
   你会看到：
   - 机器人名称
   - 头像
   - Webhook 地址
   - 安全设置
   ```

### 3.2 复制 Webhook

1. **找到 Webhook 地址**
   ```
   格式：
   https://oapi.dingtalk.com/robot/send?access_token=xxxxxxxxxxxxxxxxxxx
   ```

2. **点击「复制」按钮**
   ```
   旁边有「复制」或 📋 图标
   点击它
   ```

3. **验证已复制**
   ```
   粘贴到文本编辑器
   确认地址完整
   ```

4. **⚠️ 重要提醒**
   ```
   - 不要公开 Webhook 地址
   - 不要提交到 Git
   - 妥善保管
   ```

## 4. 配置安全设置

### 4.1 方式一：加签（推荐）⭐

#### 步骤 1：启用加签

1. **在机器人配置页面**
   ```
   找到「安全设置」区域
   ```

2. **选择「加签」**
   ```
   点击「加签」选项
   或勾选复选框
   ```

3. **查看 Secret**
   ```
   系统会生成一个 Secret：
   SECxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   
   格式：SEC + 32 位随机字符
   ```

#### 步骤 2：复制 Secret

1. **点击「复制」按钮**
   ```
   复制 Secret 到剪贴板
   ```

2. **验证已复制**
   ```
   粘贴到文本编辑器
   确认完整（包含 SEC 前缀）
   ```

#### 步骤 3：保存配置

1. **点击「保存」按钮**
   ```
   页面底部
   点击「保存」
   ```

2. **确认保存成功**
   ```
   显示提示：「保存成功」
   ✅ 加签已启用
   ```

### 4.2 方式二：自定义关键词

#### 步骤 1：设置关键词

1. **在机器人配置页面**
   ```
   找到「安全设置」区域
   ```

2. **选择「自定义关键词」**
   ```
   点击「自定义关键词」选项
   ```

3. **添加关键词**
   ```
   输入关键词，每个一行：
   Dashboard
   报警
   通知
   系统
   ```

4. **点击「保存」**
   ```
   ✅ 自定义关键词已配置
   ```

### 4.3 安全设置对比

| 方式 | 安全性 | 难度 | 推荐度 |
|------|--------|------|--------|
| **加签** | ⭐⭐⭐⭐⭐ | 中 | ⭐⭐⭐⭐⭐ |
| **关键词** | ⭐⭐⭐ | 低 | ⭐⭐⭐ |

创建 `test_dingtalk.py`：

```python
#!/usr/bin/env python3
"""
钉钉机器人测试脚本
"""

import hashlib
import hmac
import base64
import time
import json
import requests
import urllib.parse
from dotenv import load_dotenv
import os
from pathlib import Path

# 加载环境变量
env_path = Path(__file__).parent / '.env'
load_dotenv(env_path)

WEBHOOK_URL = os.getenv('DINGTALK_WEBHOOK_URL')
SECRET = os.getenv('DINGTALK_SECRET')

def gen_sign(secret):
    """生成钉钉签名"""
    timestamp = str(round(time.time() * 1000))
    secret_enc = secret.encode('utf-8')
    string_to_sign = f'{timestamp}\n{secret}'
    string_to_sign_enc = string_to_sign.encode('utf-8')
    hmac_code = hmac.new(secret_enc, string_to_sign_enc, digestmod=hashlib.sha256).digest()
    sign = urllib.parse.quote_plus(base64.b64encode(hmac_code))
    return timestamp, sign

def send_message(text, msg_type='text'):
    """发送消息到钉钉"""
    
    # 构建请求 URL
    if SECRET:
        timestamp, sign = gen_sign(SECRET)
        url = f'{WEBHOOK_URL}&timestamp={timestamp}&sign={sign}'
    else:
        url = WEBHOOK_URL
    
    # 构建消息内容
    if msg_type == 'text':
        data = {
            'msgtype': 'text',
            'text': {
                'content': text
            }
        }
    elif msg_type == 'markdown':
        data = {
            'msgtype': 'markdown',
            'markdown': {
                'title': 'Dashboard 通知',
                'text': text
            }
        }
    else:
        data = {
            'msgtype': 'text',
            'text': {'content': text}
        }
    
    try:
        response = requests.post(url, json=data, timeout=10)
        result = response.json()
        
        if result.get('errcode') == 0:
            print('✅ 发送成功！')
            return True
        else:
            print(f'❌ 发送失败：{result}')
            return False
    except Exception as e:
        print(f'❌ 请求错误：{e}')
        return False

if __name__ == '__main__':
    print('🧪 测试钉钉机器人...')
    print('')
    
    # 测试 1：简单文本消息
    print('测试 1: 发送文本消息...')
    send_message('🤖 Dashboard 测试消息', 'text')
    print('')
    
    # 测试 2：Markdown 消息
    print('测试 2: 发送 Markdown 消息...')
    markdown_text = f'''## 📊 Dashboard 测试报告

**时间**: {time.strftime('%Y-%m-%d %H:%M:%S')}

### ✅ 系统状态
- 服务运行：正常
- 数据库：正常
- 网络：正常

### 📈 今日数据
- GMV: $1,234
- 订单：45 单
- 利润：$487
'''
    send_message(markdown_text, 'markdown')
    print('')
    
    print('✅ 测试完成！请查看钉钉群消息。')
```

## 7. 集成到 Dashboard

### 7.1 创建集成模块

创建 `dingtalk_integration.js`：

```javascript
/**
 * 钉钉通知集成模块
 */

const crypto = require('crypto');
const urllib = require('url');

class DingTalkNotifier {
  constructor() {
    this.webhookUrl = process.env.DINGTALK_WEBHOOK_URL;
    this.secret = process.env.DINGTALK_SECRET;
  }

  /**
   * 生成钉钉签名
   */
  genSign(secret) {
    const timestamp = String(Date.now());
    const stringToSign = `${timestamp}\n${secret}`;
    const hmac = crypto.createHmac('sha256', stringToSign);
    hmac.update('');
    const sign = encodeURIComponent(
      hmac.digest('base64')
    );
    return { timestamp, sign };
  }

  /**
   * 构建请求 URL
   */
  buildUrl() {
    if (!this.secret) {
      return this.webhookUrl;
    }

    const { timestamp, sign } = this.genSign(this.secret);
    return `${this.webhookUrl}&timestamp=${timestamp}&sign=${sign}`;
  }

  /**
   * 发送消息
   */
  async send(message, options = {}) {
    if (!this.webhookUrl) {
      console.error('❌ 未配置 DINGTALK_WEBHOOK_URL');
      return false;
    }

    const url = this.buildUrl();
    const msgType = options.msgType || 'text';

    let data;
    if (msgType === 'text') {
      data = {
        msgtype: 'text',
        text: { content: message }
      };
    } else if (msgType === 'markdown') {
      data = {
        msgtype: 'markdown',
        markdown: {
          title: options.title || 'Dashboard 通知',
          text: message
        }
      };
    } else {
      data = {
        msgtype: 'text',
        text: { content: message }
      };
    }

    try {
      const response = await fetch(url, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(data)
      });

      const result = await response.json();

      if (result.errcode === 0) {
        console.log('✅ 钉钉通知已发送');
        return true;
      } else {
        console.error('❌ 钉钉通知失败:', result);
        return false;
      }
    } catch (error) {
      console.error('❌ 钉钉通知异常:', error);
      return false;
    }
  }

  /**
   * 发送文本消息
   */
  async sendText(text) {
    return this.send(text, { msgType: 'text' });
  }

  /**
   * 发送 Markdown 消息
   */
  async sendMarkdown(title, content) {
    return this.send(content, { msgType: 'markdown', title });
  }

  /**
   * 发送建议完成通知
   */
  async notifySuggestionCompleted(suggestion) {
    const markdown = `## ✅ 建议已完成

**标题**: ${suggestion.title}
**负责人**: ${suggestion.assignee}
**反馈**: ${suggestion.feedback || '无'}

> 时间：${new Date().toLocaleString('zh-CN')}
`;
    return this.sendMarkdown('建议完成通知', markdown);
  }

  /**
   * 发送日报
   */
  async sendDailyReport(metrics) {
    const date = new Date().toLocaleDateString('zh-CN');
    const markdown = `## 📊 Dashboard 日报

**日期**: ${date}

### 核心指标
- **GMV**: $${metrics.gmv}
- **订单**: ${metrics.orders} 单
- **利润**: $${metrics.profit}
- **ROAS**: ${metrics.roas}

> 数据来自 Dashboard 系统
`;
    return this.sendMarkdown('Dashboard 日报', markdown);
  }

  /**
   * 发送告警
   */
  async sendAlert(title, message) {
    const markdown = `## 🚨 ${title}

${message}

> ⚠️ 请立即处理！
`;
    return this.sendMarkdown(title, markdown);
  }
}

module.exports = DingTalkNotifier;
```

## 8. 故障排查

### 问题 1：发送失败

**错误信息**：
```json
{"errcode":400,"errmsg":"invalid token"}
```

**原因**：Webhook 地址错误

**解决**：
1. 检查 Webhook 是否完整复制
2. 确认 access_token 正确

### 问题 2：签名失败

**错误信息**：
```json
{"errcode":401,"errmsg":"sign not match"}
```

**原因**：
1. Secret 错误
2. 时间戳偏差

**解决**：
1. 重新复制 Secret（包含 SEC 前缀）
2. 校准服务器时间

### 问题 3：关键词不匹配

**错误信息**：
```json
{"errcode":400,"errmsg":"keyword not match"}
```

**原因**：消息内容不包含关键词

**解决**：
1. 在消息中添加至少一个关键词
2. 或修改机器人配置的关键词

### 问题 4：频率限制

**错误信息**：
```json
{"errcode":429,"errmsg":"too many requests"}
```

**原因**：超过频率限制

**解决**：
- 钉钉限制：20 条/分钟
- 降低发送频率
- 合并消息
