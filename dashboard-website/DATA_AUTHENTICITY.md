## 数据来源与验证渠道

### 1. 销售业绩数据（GMV、订单、利润）

| 数据项 | 数据来源 | 验证渠道 | 更新频率 |

| 利润 | 财务系统 | 财务报表 | 每小时 |

**验证方式：**
- 李总可随时登录 Shopify/Amazon 后台核对

- Dashboard 显示数据更新时间戳

## 数据同步架构

```
真实数据源
    ↓
OpenClaw Agent (定时抓取)
    ↓
数据清洗与验证
    ↓
SQLite 数据库 (Dashboard 数据源)
    ↓
前端展示 (每 30 秒刷新)
```

### 实现方案

**方案 A：实时 API 对接（推荐）**
```javascript
// 每 30 秒从真实系统拉取数据
async function syncRealData() {
  // Shopify API
  const shopifyData = await fetch('https://your-store.myshopify.com/admin/api/orders');
  
  // Meta Ads API
  const adsData = await fetch('https://graph.facebook.com/v18.0/act_XXX/insights');
  
  // 更新 Dashboard 数据库
  updateDashboardDB({ shopifyData, adsData });
}

setInterval(syncRealData, 30000);
```

## 数据异常处理

### 预警机制
```yaml
数据异常检测:
  - 数据超过 1 小时未更新 → 黄色预警
  - 数据超过 4 小时未更新 → 橙色预警
  - 数据超过 24 小时未更新 → 红色预警
  - 数据与真实系统差异>10% → 立即警报

处理方式:
  - 预警 → Dashboard 显示"数据可能过期"
  - 警报 → 发送消息给李总 + 安全 Agent 介入
```

### 数据校正流程
```
发现数据异常
    ↓
安全 Agent 自动检测
    ↓
暂停展示异常数据（显示"--"）
    ↓
通知相关 Agent 更新
    ↓
重新拉取真实数据
    ↓
验证数据准确性
    ↓
恢复展示 + 记录事故报告
```
