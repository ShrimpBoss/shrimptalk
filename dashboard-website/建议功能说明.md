# 💬 建议与指示功能 - 使用说明

## 🎯 功能概述

全新的建议管理系统，让你可以**按优先级提交建议**，团队**按优先级处理**，处理完成后**自动反馈总结**。

---

## ✨ 核心功能

### 1️⃣ 提交建议（3 步完成）

1. **填写建议内容** - 详细描述你的建议或指示
2. **选择优先级**：
   - 🔴 **高** - 立即处理（插队到最前）
   - 🟡 **中** - 尽快处理（默认）
   - 🔵 **低** - 有空处理
3. **选择分类和指派**：
   - 📁 分类：运营优化/产品改进/营销策略/系统功能/团队协作/其他
   - 👤 指派给：副经理/产品/营销/内容/安全运维/全体 Agent

### 2️⃣ 自动排序

建议列表**自动按优先级排序**：
- 高优先级建议永远在最上面
- 同优先级按状态排序：进行中 > 待处理 > 已完成

### 3️⃣ 进度跟踪

每条建议显示：
- 📊 **处理进度条**（0-100%）
- 🔄 **当前状态**（待处理/进行中/已完成/已驳回）
- 👤 **负责人**（指派的 Agent）

### 4️⃣ 完成反馈

建议完成后，自动在下方显示**绿色反馈框**：
- ✅ 完成标记
- 📝 总结性反馈（说明做了什么、结果如何）
- 📅 完成时间

## 🔧 技术实现

### 前端
- 优先级选择器（高/中/低）
- 分类选择器（6 个分类）
- 指派选择器（6 个 Agent）
- 自动排序（优先级 > 状态 > 时间）
- 进度条显示
- 反馈框样式

### 后端 API
- `POST /api/suggestions` - 提交建议
- `GET /api/suggestions` - 获取列表（自动排序）
- `PUT /api/suggestions/:id` - 更新状态/进度/反馈

### 数据库字段
```sql
- title: 标题
- content: 详细内容
- category: 分类
- priority: 优先级 (high/medium/low)
- status: 状态 (pending/in_progress/completed/rejected)
- assignee: 负责人
- progress: 进度 (0-100)
- feedback: 完成反馈
- created_at: 创建时间
- completed_at: 完成时间
```

## 🚀 快速开始

1. 打开 Dashboard：http://localhost:8888
2. 滚动到页面底部「💬 建议与指示」
3. 填写建议，选择优先级和分类
4. 点击「提交建议」
5. 等待团队处理和反馈！

## 💡 最佳实践

1. **优先级使用**：
   - 🔴 高：真正紧急的事（系统故障、严重 Bug）
   - 🟡 中：重要但不紧急（功能优化、体验改进）
   - 🔵 低：锦上添花（长期规划、探索性建议）

2. **内容描述**：
   - 越详细越好
   - 包含背景、问题、期望结果
   - 有数据支撑更有说服力

3. **跟进进度**：
   - 定期查看建议列表
   - 关注进度条变化
   - 完成后查看反馈总结
